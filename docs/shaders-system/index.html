12 - Shaders system

Brakeza3D offers the possibility to incorporate your own GLSL shaders.

Just like with Scripts, you can link Shaders to your scenes and/or Object3D. Shaders can be created directly using GLSL code, giving developers full control over the rendering pipeline, or through a node-based interface for a more visual and intuitive workflow.

For advanced users, it's also possible to create shaders that modify Mesh3D geometry directly, allowing procedural effects, custom vertex manipulation, or other advanced rendering techniques. This flexibility makes it possible to extend Brakeza3D beyond standard rendering and achieve unique visual results tailored to your project.

12.1) Types of Shaders​

Brakeza3D supports multiple types of shaders, allowing developers and artists to customize rendering in a variety of ways. Shaders can be applied at the scene level or to individual objects, and they can be created using GLSL code or through a visual node-based interface. This flexibility enables both visual effects for the whole scene and precise geometry manipulation for specific objects.

You can create shaders from the UI. Brakeza3D will automatically generate a .vs and .fs file (vertex shader and fragment shader, respectively), which you can modify as you wish. A JSON file with metadata will also be generated.

12.1.1) PostProcessing GLSL​

A postprocessing shader is applied after the scene has been rendered, modifying the final image before it is displayed on the screen. These shaders are ideal for effects like color grading, depth of field, fog, blur, or screen-wide visual adjustments. They can only be applied at the scene level and affect all objects within it.

12.1.2) Object GLSL​

A geometry shader processes vertices of a Mesh3D and can modify the shape, position, or other attributes of objects before rasterization. This allows for procedural geometry effects, vertex animation, or custom object transformations. Object GLSL shaders are applied only to individual meshes and do not affect the whole scene.

12.1.3) PostProcessing Shader Nodes​

These shaders offer the same effects as GLSL postprocessing shaders but use a visual node-based system. This allows artists and designers to create complex visual effects without writing code, connecting nodes for operations like blur, color adjustment, or masking.

12.1.4) Object Shader Nodes​

Similar to Object GLSL shaders, these operate on Mesh3D objects but are created through nodes instead of code. They are ideal for users who want to modify geometry, apply procedural effects, or customize vertex properties visually, while still having powerful control over individual objects.

12.2) Shader Variables (uniforms)​

You can also configure the variables (uniforms) that each shader will receive from the UI.

You can create your own variables primitives vars:

  • int
  • float
  • vec2
  • vec3
  • vec4
  • texture

Or you can select system uniforms:

  • delta_time (float)
  • execution_time (float)
  • scene (texture).

Additionally, geometry shaders (which only apply to Mesh3D) allow you to use system types such as diffuse or specular, with textures belonging to the particular model.

12.3) Postprocessing Shader Template​

Below are empty templates for the postprocessing shader.

VertexShader:

#version 330 core

layout (location = 0) in vec4 vertex; // <vec2 position, vec2 texCoords>

out vec2 TexCoords;

uniform mat4 model;
uniform mat4 projection;

void main()
{
    TexCoords = vec2(vertex.z, -vertex.w);
    gl_Position = projection * model * vec4(vertex.xy, 0.0, 1.0);
}

12.4) Geometry Shader Template​

Below are empty templates for the geometry shader.

VertexShader:

#version 330 core

layout (location = 0) in vec3 aPos;
layout (location = 1) in vec3 aNormal;
layout (location = 2) in vec2 aTexCoords;

out vec3 FragPos;
out vec3 Normal;
out vec2 TexCoords;

uniform mat4 model;
uniform mat4 view;
uniform mat4 projection;

void main()
{
    FragPos = vec3(model * vec4(aPos, 1.0));
    Normal = mat3(transpose(inverse(model))) * aNormal;
    TexCoords = aTexCoords;

    gl_Position = projection * view * vec4(FragPos, 1.0);
}

You can find the shaders that Brakeza3D uses internally in the /GLSL folder.

Line3DLine3D.vs / Line3D.fsRenders 3D lines
